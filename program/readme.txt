本工程用于Robocon2020--绿茵争锋
开发本工程需注意
1.各种相关的原文件只能相应的文件夹或源文件内书写，不得混淆
2.代码风格统一，符合代码风格规范和注释规范
3.开发前请详细阅读各文件夹内的readme了解清楚本文件夹用途
4.在各个文件夹下的readme内，罗列出当前文件包含的重要信息


代码风格规范
1.宏定义需全部大写
2.全局变量加上前缀global_
3.静态变量前加上static_
4.形参小写
5.使用英文命名，不得使用拼音

注释规范：
在函数的前方书写以下文本
/**
*@function 函数名
*@param    形参名及其意义
*@brief    函数功能简介
*@retval   返回值
**/

额外要求
1.在每个c文件开头需书写如下信息
/**
*@file    文件名
*@author  作者名
*@version 版本号
*@date    最后修改日期
*@brief   文件功能
**/
